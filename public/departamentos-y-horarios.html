📊 Resumen de Origen de Datos

  • Departamentos/Áreas: Se gestionan desde Firebase Firestore (colección areas)
  • Horarios por Departamento: Se buscan primero en Firestore (colección mealSchedules), si no existen se usa el código como fallback
  • Grupos de Empleados: Se buscan primero en Firestore (colección employeeGroups), si no existen se usa el código como fallback
  • Colecciones en Firebase:
    • areas - Departamentos/Áreas
    • mealSchedules - Horarios fijos por departamento+turno
    • employeeGroups - Grupos de empleados con horario específico
📝 Nota Importante
Los datos se cargan desde Firestore. Mientras no haya datos en Firestore, se usan los valores de STATIC_CONFIG en el código como fallback. Desde el panel de administración en "Ajustes → Horarios de Comida", puedes:
  • Agregar/editar/eliminar horarios por departamento
  • Agregar/editar/eliminar grupos de empleados
  • Inicializar desde los datos predeterminados del código
